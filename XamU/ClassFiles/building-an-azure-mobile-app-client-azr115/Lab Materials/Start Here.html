Before you begin

This is a cross-platform class which uses Xamarin.Forms to to target iOS, Android or Windows. To complete the coding exercises, you will need a Xamarin development environment with either an emulator/simulator or a device to run the exercise on. Please read the setup requirements if you need help installing the Xamarin development environment.

This class assumes you have already setup an Azure account and have a basic knowledge of the Azure Management portal. If you have not done anything with Azure yet, then please review the materials from AZR101 - Introduction to Azure.

Overview

This class focuses on the client-side model for building a Xamarin application that uses an Azure App Service mobile back-end. We will explore the techniques used to add support to an existing Xamarin application to consume data from an Azure service, how to support offline caching and synchronization, and how to manage conflicts when they arise.

Lecture

AZR115 - Building an Azure Mobile App Client.

Objectives

  1. Connect your mobile app to Azure
  2. Access table data from Azure
  3. Add support for offline synchronization

Exercises

  1. Add Azure support to our Xamarin application
    Exercise
    You will start by adding support to connect to an existing Xamarin.Forms application to an Azure App Service mobile app.
  2. Customize the DTOs for the Survey service
    Exercise
    Next, you will annotate the Data Transfer Objects (DTOs) in the app to properly match the shape of the service data being interacted with.
  3. Fill in the logic to query and update our survey records
    Exercise
    In this exercise, you will complete the logic to read and update the survey response records in your application.
  4. Add support to our app for offline data caching
    Exercise
    In this exercise, you will change your application to support offline synchronization.
  5. Synchronizing to the remote database
    Exercise
    In this exercise, you will add the code necessary to push and pull changes from the remote DB to the local DB and back.
  6. Add error recovery code to support conflicts
    Homework Exercise
    In this homework exercise, you will add code into the survey application to handle conflict errors that occur during synchronization.

Model-View-ViewModel

For fun, there is a completed version of the entire lab, but written using the Model-View-ViewModel pattern to demonstrate how to deal with asynchronous collection loading, busy indicators, passing data between screens, and integrating Azure services into your MVVM apps. If you are interested in MVVM feel free to explore the sample after class; it's contained in the MVVM folder.
