Resources and Styles

Duration

10 minutes

Goals

You will use Style inheritance to avoid repeated Setters in Styles that target the same type of control.

Required assets

The provided Resources folder for this exercise contains a subfolder named Complete with a solution you can use to check your work.

This lab is a continuation of the previous one. If you did not complete the previous exercise, you can use the Completed solution from the previous exercise as starter code for this exercise.

Challenge

You can use the high-level description here to work through the lab on your own. Alternatively, you can use the instructions below if you would like step-by-step guidance.

  • Open the StandardTipPage.xaml file in the TipCalculator project.
  • Define a Style for a Label that sets the FontSize to 22.
  • Apply the new Style to the two labels on the page that use a font size of 22.
  • Use Style inheritance to refactor your two Styles and eliminate the repeated Setter.

Steps

Below are the step-by-step instructions to complete the exercise.

Repeated Setters

Here you will define a new Style that will overlap the one you wrote for the previous exercise. The goal here is to motivate the use of Style inheritance by showing how Setters can end up as repeated code.

  1. Open the TipCalculator project.
  2. Open StandardTipPage.xaml.
  3. Add a Style to the page's resource dictionary. Use a Key of baseLabelStyle and a TargetType of Label.
  4. Add a Setter to your style that sets the FontSize property to 22. Notice how this Setter also appears in your other Style.
  5. Apply the baseLabelStyle to the two labels on the page that display the output (i.e. that display the calculated amounts for the tip and the total). Remove the explicit FontSize setting from these labels.
  6. Run the app on a platform of your choice. Verify that the UI looks the same as it did before you replaced the property settings with a Style.

Use Style inheritance

This part asks you to refactor your two Styles into an inheritance hierarchy in order to eliminate the repeated Setter.

  1. Open StandardTipPage.xaml.
  2. Locate the infoLabelStyle Style in the page's resource dictionary. Set the BasedOn property of this style to your baseLabelStyle.

    3. Show Code 

    <Style x:Key="infoLabelStyle" BasedOn="{StaticResource baseLabelStyle}" TargetType="Label">
  ...
</Style>

  3. Remove the Setter for FontSize from the infoLabelStyle. It is no longer needed because this Style now inherits the FontSize setting from its base Style.
  4. Run the app on a platform of your choice. Verify that the UI looks the same.

Summary

Here you used Style inheritance to eliminate repeated code from two Styles. This example was intentionally minimal so you could concentrate on the new syntax; however, the benefits become even greater in complex cases where you can use this technique to eliminate multiple repeated Setters.
