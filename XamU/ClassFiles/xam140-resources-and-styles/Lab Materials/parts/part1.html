Resources and Styles

Duration

10 minutes

Goals

This lab will motivate the use of Resources and Styles. You will examine a pre-written app and find examples of repeated XAML and C# code.

The provided app implements a Tip Calculator you might use at a restaurant. It has two pages: a standard calculator that always produces a 15% tip and a custom calculator that lets you control the calculation. The screen shots below show the two pages of the app running on iOS (the colors shown here represent the final app, after you have adjusted the colors in a subsequent exercise).

Required assets

The provided Resources folder for this exercise contains a subfolder named Start that contains the starting code for the app you will be working with.

Challenge

You can use the high-level description here to work through the lab on your own. Alternatively, you can use the instructions below if you would like step-by-step guidance.

  • Open the provided TipCalculator project.
  • Locate at least three places in the XAML where hard coded values for colors and fonts are used. Find at least two examples where these hard coded values are repeated.
  • Examine the code-behind in StandardTipPage.xaml.cs and note the repeated code used to switch between the "light" and "dark" color schemes.

Steps

Below are the step-by-step instructions to complete the exercise. You will inspect the source code of a provided project. There is no code to write in this exercise.

Find examples of repeated XAML

Here you will locate several places where repeated XAML values are used. The goal is to motivate why Resources and Styles are useful (since they let you refactor your XAML to eliminate repeated values).

  1. Open the provided TipCalculator project.
  2. Run the app on any platform you prefer. Take a few minutes to work with the app so you understand how it behaves.
  3. Open StandardTipPage.xaml.
  4. Find the code that sets the background color of the root grid. Notice that a hard coded value is used.
  5. Find the code that sets the "left column" labels to navy/22/bold. Notice that these same values are used on three labels.
  6. Find the code that sets the "right column" labels to navy/22. Notice that these same values are used on two labels.
  7. Notice that some of the property settings seem to form a logical group. For example, the combination of navy/22 is used on multiple labels.
  8. Consider the difficulty of modifying the FontSize value. You would need to change it in five places which is error prone.

Find examples of repeated code

The provided project implements a simplified version of "light" and "dark" color schemes for the StandardTipPage. Here, you will examine how this is coded.

  1. Open StandardTipPage.xaml.cs.
  2. Locate the two click handlers that update the UI colors.
  3. Notice how the code must update the colors for each control individually resulting in repeated code.

Summary

Here you observed how difficult it is to maintain an app that contains repeated code. Subsequent exercises will use Resources and Styles to eliminate much of the repeated XAML.
