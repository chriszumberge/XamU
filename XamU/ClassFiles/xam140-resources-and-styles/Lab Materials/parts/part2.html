Resources and Styles

Duration

10 minutes

Goals

In this exercise, you will use page-level Resources to eliminate repeated values in XAML.

Required assets

Please use the provided solution in the Part 01 Resources/Start folder as starter code for this exercise (i.e. the starter code for this exercise is in the folder for the previous one). The Part 02 Resources/Completed folder contains a solution you can use to check your work.

Challenge

You can use the high-level description here to work through the lab on your own. Alternatively, you can use the more detailed instructions below if you would like step-by-step guidance.

  • Open the StandardTipPage.xaml file in the TipCalculator project.
  • Define two Color resources in the page's Resources dictionary:
    • One for the background color (#c0c0c0).
    • Another for the foreground color (#606060).
    You can pick whatever colors you like, just make sure they are contrasting. You should avoid using Black and White directly as some operating systems (particularly Windows) uses these for hover effects and will make your elements disappear.
  • Use the StaticResource markup extension to apply the resources every place on the XAML page where those colors are used.

Steps

Below are the step-by-step instructions to complete the exercise.

Define Resources

Here you will define two Resources for use on a XAML page.

  1. Open the TipCalculator project.
  2. Open StandardTipPage.xaml.
  3. Create a ResourceDictionary object in the page's Resources property. The code is provided below if you need it.

    4. Show Code

    <Page.Resources>
  <ResourceDictionary>
  </ResourceDictionary>
</Page.Resources>

  4. Define a Color resource inside the resource dictionary. Give it an x:Key of "bgColor" and a value of #c0c0c0. The code is provided below if you need it.

    5. Show Code

    <Color x:Key="bgColor">#c0c0c0</Color>

  5. Define a second Color resource inside the resource dictionary. Give it an x:Key of "fgColor" and a value of #606060.

Use Resources

Here, you will apply the Resources you defined in the last section.

  1. Use the StaticResource markup extension to apply the bgColor resource to the Background property of the page's root Grid.

    2. Show Code

    <Grid x:Name ="LayoutRoot" BackgroundColor="{StaticResource bgColor}" Padding="10">

  2. Use the StaticResource markup extension to apply the fgColor resource to the TextColor property of all the Labels that use a TextColor of Navy.
  3. Run the application on a platform of your choice and verify that the StandardTipPage displays dark text on a light background at startup.

Summary

Here you replaced the hard coded colors in a XAML page with Resources. The colors are now defined in one place instead of being repeated so they are easier to update when needed.
