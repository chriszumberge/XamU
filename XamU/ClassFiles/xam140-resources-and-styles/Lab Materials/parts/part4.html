Resources and Styles

Duration

10 minutes

Goals

In this lab, you will define and apply a page-level Style.

Required assets

The provided Resources folder for this exercise contains a subfolder named Complete with a solution you can use to check your work.

This lab is a continuation of the previous one. If you did not complete the previous exercise, you can use the Completed solution from the previous exercise as starter code for this exercise.

Challenge

You can use the high-level description here to work through the lab on your own. Alternatively, you can use more detailed instructions below if you would like step-by-step guidance.

  • Open the StandardTipPage.xaml file in the TipCalculator project.
  • Define a Style for a Label that sets the FontSize to 22 and the FontAttributes to Bold.
  • Apply the Style to all Labels on the page that use the combination 22/Bold.

Steps

Below are the step-by-step instructions to complete the exercise.

Define a Style

Here you will implement a Style and store it in a page-level dictionary.

  1. Open the TipCalculator project.
  2. Open StandardTipPage.xaml.
  3. Add a Style to the page's resource dictionary. Use a Key of infoLabelStyle and a TargetType of Label.

    4. Show Code 

    <Style x:Key="infoLabelStyle" TargetType="Label">
</Style>

  4. Add a Setter to your style that sets the FontSize property to 22.

    5. Show Code 

    <Setter Property="FontSize" Value="22" />

  5. Add a Setter to your style that sets the FontAttributes property to Bold.

Apply a Style

Here, you will apply your Style to a few XAML elements.

  1. Open StandardTipPage.xaml.
  2. Locate the three Labels that use TextSize of 22 and FontAttributes of Bold. Remove those property assignments from the labels.
  3. Use the StaticResource markup extension to assign the Style you just defined to the Style property of the three Labels. The sample code for one Label is provided below if you need it.

    4. Show Code 

    <Label Style="{StaticResource infoLabelStyle}" ... />

  4. Run the app on a platform of your choice. Verify that the UI looks the same as it did before you replaced the property settings with a Style.

Summary

Here you replaced individual property settings with a Style. This reduced the clutter of multiple property settings on each control and made the XAML easier to read. It also made it easier to achieve consistency across your UI since property settings are grouped together in a central location.
